# 系统要求（部署前先看这个）

> 适用：BrightStar 培训助手（AWS Serverless 微信 agent 版）。
> 更新：2026-06　区域：东京 `ap-northeast-1`（JST）

BrightStar 主体是 **AWS Serverless**，**本身没有服务器、没有显存要求**。真正的“硬件/资源”要求集中在两处：① AWS 账号侧 ② 一台做“可信IP 中转”的小主机。最后给出**可选**的“本地大模型”路径的显存要求（当前用 Bedrock 则用不到）。

---

## 1. AWS 账号侧（必需）
| 项 | 要求 |
|---|---|
| AWS 账号 | 一个，能创建 Lambda / API Gateway / DynamoDB / EventBridge / Budgets |
| 区域 | **ap-northeast-1（东京）** |
| 部署权限 | 部署者 IAM 需 CloudFormation + 上述服务的创建权限 |
| **Bedrock 模型访问** | 在 Bedrock 控制台开通：`amazon.titan-embed-text-v2`（向量）、`jp.anthropic.claude-haiku-4-5`（生成，日本境内 profile）；**Anthropic 系模型须先提交一次性「use case details」用例表** |
| 成本 | 几乎全部按需/免费额度内：DynamoDB 按需、Lambda/HTTP API 调用计费、Bedrock 按 token（一次 AI 问答约 0.5 日元）。建议建 Budgets 月度告警 |

> 无固定月费、零流量零成本。唯一“地板价”陷阱是 OpenSearch——本系统**不用**它。

## 2. 中转主机（必需，用于微信客服 kf 出站“可信IP”）
企业微信客服 API 要求出站 IP 在“可信IP”白名单内，所以需要一台 IP 固定、能被 Lambda 入站访问的小主机做中转。

| 项 | 要求 | 说明 |
|---|---|---|
| OS | Linux（当前 Ubuntu 24.04） | |
| Python | 3.x + `requests` | 跑 `relay/wechat_relay.py` |
| CPU/内存 | **极低**（1 核 / 512MB 足矣） | 只是一个轻量 HTTP 转发 |
| **GPU/显存** | **不需要** | 中转不涉及模型 |
| 出站网络 | 能访问 `qyapi.weixin.qq.com` | 调企业微信 API |
| 入站可达 | Lambda 能连到它 | 当前用 **Tailscale Funnel**（无需公网端口转发）；其公网出口 IP 需加进企业微信可信IP |
| 可信IP | 该主机公网出口 IP 加入企业微信“可信企业IP” | IP 变更需重新加白名单 |

> 当前部署：中转跑在 sons02（`10.32.1.41`），经 `https://sons02-relay.tail5a0084.ts.net` 暴露，出口 IP `111.104.33.44` 已在白名单。开机自启用 crontab；可换 systemd。

## 3. 客户端 / 第三方
- **学员**：微信（与“BrightStar 研修アシスタント”客服号对话）。
- **网站访客**：任意现代浏览器（课程网站，登录码登录）。
- **老师/运营**：企业微信（发文字命令）；企业微信管理后台（配置回调、可信IP）。
- **企业微信**：CorpID / 应用 AgentId+Secret / 客服号 open_kfid / Token+EncodingAESKey。

## 5.（可选）改用“本地大模型”而非 Bedrock —— 这时才有显存要求
当前 AI 问答用 **Bedrock**（无需本地 GPU、数据留日本）。如果将来想把生成换成**本机自建大模型**（llama.cpp），则需要 GPU：

| 项 | 要求（参考 sons02 实测环境） |
|---|---|
| GPU | NVIDIA，架构与显存按模型定；如 RTX PRO 6000 Blackwell **96GB**（sm_120） |
| CUDA | Toolkit ≥ 12.8（Blackwell sm_120 必需），与驱动匹配 |
| 显存估算 | Qwen3-32B Q8 ≈ 34GB 权重 + 64k 上下文 KV ≈ 10GB ≈ **45–48GB**；与其他占用（如 ComfyUI ~17GB）共存需留足 |
| 编译 | cmake + `-DGGML_CUDA=ON -DCMAKE_CUDA_ARCHITECTURES=120` |

> ⚠️ 这是**另一条可选路线**，不属于当前线上架构。当前用 Bedrock 就**完全不需要 GPU/显存**。是否切换需评估：本地模型省 token 费但要自建运维、且 Lambda 要经中转访问本地服务。
