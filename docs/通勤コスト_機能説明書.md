# 要員通勤コスト比較ツール 機能説明書

本書は、本ツールの機能・入出力仕様・制約をまとめたものです。

---

## 1. 取得する 4 指標

1 経路あたり、以下を取得します。

| 指標 | 内容 |
|------|------|
| 所要時間 | 出発駅→到着駅の所要分 |
| 乗換回数 | 経路上の乗換回数 |
| 片道運賃 | IC 運賃・きっぷ運賃の 2 種 |
| 通勤定期代 | 1ヶ月・3ヶ月・6ヶ月（定期種別は「通勤」） |

補助情報として **経由（乗換駅を含む経路）** と **走行距離** も取得します。

---

## 2. 経路選択の戦略

複数の経路候補が返るため、**先頭を盲目的に採らず**明示ルールで 1 経路に絞ります。

| 戦略 | ルール |
|------|--------|
| cheapest（既定） | 通勤定期代（1ヶ月）が最安。同額なら所要時間が短い方 |
| fastest | 所要時間が最短。同時間なら定期代が安い方 |

---

## 3. 一括比較

- 現場（到着駅）を指定すると、**status=available（待機中）** の全要員について通勤コストを取得し、戦略順に並べます。
- **status=assigned（現場確定済み）** の要員は対象から自動除外します。
- 同一の (出発駅, 到着駅, 戦略) は実行内メモとキャッシュで **1 回しか問い合わせません**。

---

## 4. 入力（駅名）の正規化

現場の駅名は表記がゆれても解決できるよう、段階的に正規化します。

| 入力例 | 対応 |
|--------|------|
| 東京 / 渋谷（漢字） | そのまま解決 |
| とうきょう / シブヤ（かな） | 読みで解決 |
| tokyo（ローマ字） | 読みで解決 |
| 东京 / 涩谷（簡体字中国語） | AI（Bedrock）で日本語駅名へ正規化してから解決 |

---

## 5. 住所 → 最寄駅

社員情報が住所しか無い場合、住所をジオコーディングし、近傍の鉄道駅から **最寄駅** を判定します。
**保存するのは駅名のみで、住所は保存しません**（個人情報配慮）。

---

## 6. 社員データ項目

| 項目 | 内容 |
|------|------|
| staff_id | 社員 ID（社内システムの花名册と同一キー） |
| name | 氏名 |
| nearest_station | 最寄駅（住所ではなく駅名のみ保持） |
| status | available（待機）／ assigned（現場確定） |
| assigned_site | 確定現場名（assigned のとき） |
| department / updated_at | 部署 / 更新時刻 |

---

## 7. 出力

- **LINE**：テキストで「現場名・待機中人数・各要員の時間/乗換/運賃/定期・経由」を返信（合計や平均は表示しません）。
- **コマンドライン**：端末表示に加え CSV 出力に対応。
- **API**：JSON（および CSV）で返却。

---

## 8. 既知の制約

- 取得値は **取得時点のダイヤ** に依存し変動します。最終確認は公式の運賃案内で行ってください。
- 暫定のデータ取得元を利用しており、利用は低頻度・キャッシュ前提です。
- 簡体字中国語は AI 変換に依存するため、まれに解決できない駅があります（日本語駅名で再送）。
