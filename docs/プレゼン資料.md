# BrightStar 社内システム ｜ ご紹介資料（プレゼン）

> LINE だけで「勤怠・通勤費」の提出と人事管理を完結。フルサーバーレス × AWS CDK。

---

### 1. 表紙
**BrightStar 社内システム** — 勤怠・通勤費の提出を、LINE のチャットだけで。
受託開発ケーススタディ／AWS サーバーレス × LINE。

### 2. 課題
- 毎月の勤怠表・通勤費精算が、メール添付・紙・表計算でバラバラ。
- 「誰が出していないか」が見えず、人事が一人ずつ催促。
- 専用アプリ導入・ID 配布は重い。社員は**普段使う LINE**で済ませたい。

### 3. ソリューション
- **社員**：記入した Excel を LINE に送るだけで提出（様式も LINE で取得、再提出可、自分の履歴も確認）。
- **人事**：LINE で全社の提出状況を即確認、未提出者へワンタップ督促、当月分を一括ダウンロード。
- **名簿主導**：社員名簿（DB）が「提出対象」を決定。未登録者も一覧に `※Line未登録` で可視化。

### 4. 主要機能（一覧）
| 社員 | 人事 |
|---|---|
| 提出（再提出可）／テンプレ／履歴(DL) | 未提出確認／一覧／催促／一括DL |
| 氏名だけで登録（部署は名簿から） | 社員名簿の追加・変更・削除（変更は人事全員へ通知） |

### 5. 自動リマインド
- 既定：毎月 25・28 日 09:00（JST）に**未提出者だけ**へ自動配信。
- 人事が手動でも即時督促。提出済みは自動的に対象外。

### 5.5 提出時の内容チェック（自動）
提出された Excel を自動検証し、誤りを未然に防止。
- **年月の照合**（拒否）：経費=A1「YYYY年M月分」／勤怠=B5 の日付が提出月と一致しなければアップロード不可。
- **氏名の照合**（拒否・勤怠）：B3 の氏名が登録氏名と不一致なら不可。
- **休日勤務の注意**（警告・勤怠）：土日・祝日に勤務時間があれば、該当日を挙げて確認を促す。
- **日付の欠落**（警告・勤怠）：5 行目の日付が 1 日〜月末まで揃っていなければ通知。
- 解析は標準ライブラリのみ（依存ゼロ／ふりがな除外／日付シリアル対応）。

### 6. アーキテクチャ（フルサーバーレス）
```
LINE ─► Lambda Function URL ─► webhook(Python) ─► DynamoDB(roster/employees/submissions) + S3
EventBridge(cron) ─► reminder ─► LINE push（未提出者）   SSM(SecureString): LINE 資格情報
```
- 固定サーバなし・従量課金・自動スケール。LINE は署名検証のみで接続（中継・暗号不要）。
- IaC は **AWS CDK (TypeScript)**、Lambda は **Python 3.12/arm64・外部依存ゼロ**。

### 7. セキュリティ・データ保護
- 資格情報は SSM SecureString（git に出さない）。S3 は非公開＋暗号化＋バージョン管理。
- ダウンロードは **HMAC 署名付き短縮リンク**で越権防止。
- 保管：2 か月は即時DL、以降 Deep Archive（最安）、1 年で自動削除。

### 8. 効果
- 提出・督促・集計の**工数を大幅削減**、未提出の取りこぼしゼロへ。
- 専用アプリ不要（LINE）。**固定費ほぼゼロ**の従量運用。
- CDK 管理で**変更・横展開が容易**（事業所追加・項目追加など）。

### 9. 拡張ロードマップ（任意）
- Excel 中身の自動集計（工数・金額）／配信時刻のセルフ設定／他チャネル展開／多言語強化。
