# BrightStar 社内システム ｜ 機能説明書

毎月の「作業時間記録簿（勤怠）」と「経費（通勤費）」を **LINE のチャットだけで提出・管理** する社内向けシステムです。社員は LINE で提出、人事は LINE で全社の提出状況確認・未提出者の督促・一括ダウンロードまで完結します。

- リージョン：AWS 東京（ap-northeast-1）
- フロント：LINE 公式アカウント（人事専用）
- バックエンド：フルサーバーレス（Lambda / DynamoDB / S3）、IaC は **AWS CDK (TypeScript)**

---

## 1. 対象書類（毎月・固定テンプレート）
| 種別 | 内部キー | 様式ファイル |
|---|---|---|
| 勤怠 | `kintai` | 作業時間記録簿（氏名）_YYYYMM.xlsx |
| 通勤費 | `commute` | 経費（氏名）_YYYYMM.xlsx |

様式は Bot から「テンプレ」でいつでも取得できます。提出は記入済み Excel を**チャットに送るだけ**。

## 1.5 提出時の自動チェック（アップロード前後）
提出された Excel の内容を自動検証します。

| 種別 | チェック | NG 時 |
|---|---|---|
| 共通 | 年月（経費=A1「YYYY年M月分」/ 勤怠=B5 の日付）が**提出月（当月）と一致**するか | アップロード拒否＋理由を通知 |
| 勤怠 | **B3 の氏名**が登録氏名（社員名簿）と一致するか | アップロード拒否＋「氏名が一致しません」 |
| 勤怠 | **休日（土日・日本の祝日）に勤務時間（6/7 行）が入力**されていないか | アップロードは許可。該当日を挙げて「お間違いなければそのまま、誤りなら再提出」と注意通知 |

- 年月・氏名の不一致は**保存されません**（修正して再提出）。
- 休日勤務の検出は**警告のみ**（休日出勤が正しい場合もあるため保存はします）。
- 祝日は土日に加え、日本の国民の祝日（ハッピーマンデー・春分/秋分・振替休日を含む）を自動判定。
- xlsx の解析は標準ライブラリのみ（依存ゼロ／ふりがな rPh は除外／日付シリアル対応）。

## 2. 社員向け機能
- **登録（初回のみ・氏名だけ）**：氏名を送信 → 社員名簿と照合して本人を特定（部署は名簿から自動取得）。名簿に無い場合は人事へ連絡を案内。
- **提出（再提出可）**：勤怠／経費の Excel を送信 → ファイル名から自動分類 → S3 保存＋受付記録。判別不能時は種別を質問。
- **テンプレ**：空白様式（勤怠・経費）をボタンからダウンロード。
- **履歴**：自分の提出履歴を一覧表示＋各ファイルのダウンロードボタン。

## 3. 人事向け機能（人事ロールのみ）
- **未提出確認[YYYYMM]**：当月（省略時）または指定月の**未提出者**を一覧。LINE 未登録の社員も表示し `※Line未登録` を付記。
- **一覧[YYYYMM]**：全社の提出状況（未提出を上段・提出済を下段）。
- **催促[YYYYMM]**：未提出者へリマインドを LINE 配信（提出済は対象外）。
- **一括DL**：当月の全提出を 1 つの zip にまとめてダウンロード。
- **社員名簿 CRUD**：`名簿` / `社員追加` / `社員変更` / `社員削除`。**変更は人事全員へ自動通知**。

## 4. リマインド（督促）
- **自動**：EventBridge スケジュール（既定：毎月 25・28 日 09:00 JST）→ 未提出者へ自動配信。
- **手動**：人事が「催促」を送信 → 即時配信。
- **重複なし**：提出済みの社員は自動的に対象外（提出＝記録更新で差分から除外）。
- **未登録者**：LINE 未登録の未提出者は配信できないため、一覧に `※Line未登録` を表示して人事が個別連絡。

## 5. 再提出時の人事確認（締め後）
月をまたいで（＝前月分などを）**再提出**した場合、**人事全員へ確認通知**を自動配信します。
> 例：「⚠️【再提出確認】山田太郎 さんが 2026-05 の勤怠を再提出しました。内容のご確認をお願いします。」

## 6. 権限管理
- **社員（employee）**：自分の提出・履歴のみ。
- **人事（hr）**：全社データの閲覧・督促・名簿管理。
- ロールは**社員名簿（roster）の `role`** で決定（LINE のグループ機能ではなく DB で管理）。

## 7. データ保管とライフサイクル
- 提出物は S3 に保存（パス：`hr/{年}/{月}/worktimes|expenses/…`）。
- **バージョン管理 ON**（再提出しても旧版を保持）。
- **2 か月間は標準ストレージ＝即時ダウンロード可**、2 か月経過後は **Glacier Deep Archive（最安）** へ自動移行、**1 年後に自動削除**。
