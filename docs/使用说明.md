# 使用说明（部署 / 运维 / 排错）

> 配合《系统要求.md》先确认前置。命令默认在仓库 `brightstar/` 目录下执行，区域 `ap-northeast-1`。
> Windows 上所有 AWS/Python 命令前都要带 `PYTHONUTF8=1`。

---

## 一、部署

### 方式 A：SAM（当前线上方式，本机无需 SAM CLI/Docker）
原理：`aws cloudformation package` 把 `src/` 打包传 S3 → `deploy` 由 CloudFormation 服务端跑 SAM 转换。本项目无第三方依赖，打包即压缩 src/。

```bash
# 1) 打包
PYTHONUTF8=1 aws cloudformation package \
  --template-file template.yaml \
  --s3-bucket brightstar-artifacts-<account>-apne1 \
  --output-template-file packaged.yaml --region ap-northeast-1

# 2) 部署（机密走 --parameter-overrides，不入库；未列出的参数保留上次值）
PYTHONUTF8=1 aws cloudformation deploy \
  --template-file packaged.yaml --stack-name brightstar-dev \
  --capabilities CAPABILITY_IAM --region ap-northeast-1 \
  --parameter-overrides Stage=dev \
    WeComRelayUrl=https://sons02-relay.tail5a0084.ts.net \
    WeComRelayAuth=**** BedrockChatModelId=jp.anthropic.claude-haiku-4-5-20251001-v1:0 \
  --tags Project=brightstar ManagedBy=sam Stage=dev
```

### 方式 B：CDK（见 `cdk/README.md`）
```bash
cd cdk && python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cdk bootstrap aws://<account>/ap-northeast-1
cdk deploy -c stage=cdk -c weComCorpId=... -c weComRelayUrl=... -c weComSecret=...
```
> CDK 默认 `stage=cdk`，与线上 SAM `dev` **独立不撞名**。勿同时用两套管同一批资源。

### 部署后必做
1. **Bedrock**：控制台（东京区）→ Model access → 开通 Titan + Anthropic Claude，**提交 Anthropic 用例表**，等几分钟生效。
2. **企业微信回调**：把输出的 `WeChatWebhookUrl` 填到企业微信（自建应用/客服回调），Token/EncodingAESKey 用部署时的值。
3. **可信IP**：把中转主机出口 IP 加进“可信企业IP”。
4. **灌 RAG 知识库**：
   ```bash
   AWS_DEFAULT_REGION=ap-northeast-1 python tools/seed_knowledge.py   # SAM(默认表名)
   # CDK 自动表名：再加 KNOWLEDGE_TABLE=<KnowledgeTableName 输出>
   ```
5. **预算告警（可选）**：建 Budgets 限定服务 “Amazon Bedrock”，阈值 $5 邮件提醒。

### 跑测试（改代码后）
```bash
PYTHONUTF8=1 python tests/test_flow.py        # 14 项端到端流程（内存假 DynamoDB）
```

## 二、运维

### 改配置 / 切模型
- 改环境变量/参数后重新 `deploy`（SAM 未列参数会保留上次值，机密无需每次重传）。
- **省钱切模型**：`BedrockChatModelId` 从 `jp...sonnet-4-5`（质量高）↔ `jp...haiku-4-5`（省钱 ~1/3），重部署即可。

### 看日志
```bash
# PowerShell（Git Bash 会把 /aws/lambda 路径搞乱）
aws logs tail /aws/lambda/brightstar-dev-webhook --since 30m --region ap-northeast-1
# 另两个：brightstar-dev-web、brightstar-dev-reminder
```

### 中转主机（sons02）
- 进程：`pgrep -af wechat_relay.py`；日志 `~/wechat-relay/relay.log`。
- Tailscale：`tailscale funnel status`（确认 `https://...ts.net` 在线）。
- 重启中转：`bash ~/wechat-relay/start.sh`（或 systemd：`sudo systemctl restart wechat-relay`）。
- 开机自启：crontab `@reboot`（或 `sudo bash ~/wechat-relay/setup.sh` 换 systemd）。
- **IP 监控**：`~/ip-monitor/ip.log`（每 10 分钟），出口 IP 若变需重新加可信IP。

### 凭证 / 口令
- 老师认证口令 = 参数 `TeacherSignupCode`（当前 `BS-TEACHER-2026`），改值重部署。
- 登录码有效期 = `LOGIN_CODE_TTL_DAYS`（7 天）。
- **所有机密只走部署参数 / context，绝不入库**；泄露要轮换。

## 三、排错（常见问题）

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 微信回调验签失败 / 配置不通过 | Token/AESKey 不一致、URL 错 | 核对企业微信里与部署参数一致；GET 验签由 webhook 回 echostr |
| kf 调用报 `errcode 60020` | 出口 IP 不在可信IP | 把中转出口 IP 加进“可信企业IP”；确认 Lambda 经中转出站 |
| 回复很慢 / 偶发重复 | 冷启动 / 重试 | 已用 msgid 去重 + 中转 keep-alive；持续慢看中转日志延迟 |
| AI 问答不回（回“暂时没能回答”） | Bedrock 用例表未提交 / IAM 缺 `bedrock:InvokeModel` / 模型 ID 错 | 提交 Anthropic 用例表；确认 Lambda 角色有 InvokeModel；模型用 `jp.` profile（新模型按需不支持，必须 inference profile） |
| 登录网站“登录码无效” | 超 7 天过期 | 微信重发「登录码」拿新的 |
| 建了课学员看不到 | 忘了发布 | `发布 课程名` |
| 提醒发不出 | 学员 48h 内无互动（微信平台限制） | 无法绕过；引导学员先互动 |
| 中转连不通 | Funnel 掉线 / 主机重启未自启 | `tailscale funnel status`、`start.sh`；查 crontab/systemd |
| 部署报 UTF-8/编码错（Windows） | 没设 `PYTHONUTF8=1` | 命令前加 `PYTHONUTF8=1 PYTHONIOENCODING=utf-8` |
| `capacity` 相关条件写入失败 | DynamoDB 保留字 | 已用 `#cap` 别名（代码层已处理） |

### 快速健康检查
```bash
# webhook Lambda 能加载（GET 无签名应回 403 invalid signature）
echo '{"requestContext":{"http":{"method":"GET"}},"queryStringParameters":{}}' > ev.json
aws lambda invoke --function-name brightstar-dev-webhook --region ap-northeast-1 \
  --payload fileb://ev.json out.json && cat out.json
# 中转公网（带共享密钥应回 400 缺字段；无密钥 401）
curl -s -X POST https://sons02-relay.tail5a0084.ts.net/ -H "X-Relay-Auth: ****" -d '{}'
```

## 四、回退 / 拆除
- 回退中转到旧云 VM：把 `WeComRelayUrl` 改回旧地址重部署。
- 拆除栈：SAM `aws cloudformation delete-stack --stack-name brightstar-dev`；CDK `cdk destroy`。**DynamoDB 默认 RETAIN（防误删数据），需手动删表**。删前务必确认数据已不需要。
