# BrightStar 社内システム ｜ コードロジック

Lambda コード（Python、外部依存ゼロ）と CDK（TypeScript）の構成・主要処理フロー。

```
brightstar-hr/
  cdk/                       # CDK(TypeScript)
    bin/app.ts               # App / Stack 起動（stage）
    lib/brightstar-hr-stack.ts  # 全リソース定義
  lambda/                    # Python 3.12（標準ライブラリ + boto3 のみ）
    common/
      config.py    # 環境変数・定数・SSM からの LINE 資格情報（遅延取得+キャッシュ）
      db.py        # DynamoDB テーブルハンドル（roster/employees/submissions）
      line.py      # LINE 適配：署名検証 / イベント解析 / reply・push / 文件DL / buttons・carousel
      messaging.py # 送信の統一出口（現状 LINE push）
      s3util.py    # 提出物の保存/署名付きDL/暫定置き/月次zip
      i18n.py      # 文言（日本語既定・中国語併記）
      business.py  # 業務ロジック（名簿・登録・提出・差分・CRUD）
    handlers/
      line_webhook.py  # 入口：GET /dl・POST webhook のルーティング
      reminder.py      # 定時/手動リマインド
  templates/  作業時間記録簿.xlsx / 経費.xlsx   # 空白様式（S3 hr/template/ へ配備）
```

---

## 入口：`handlers/line_webhook.py`
```
handler(event):
  if method == GET:  return _handle_download(event)   # /dl?type= / /dl?key=&sig=
  verify_signature(body, X-Line-Signature)  → NG: 403
  base = "https://" + host
  for ev in line.parse_events(body): _route(ev, base)
  return 200
```
- `_handle_download`：`type` は様式（hr/template/…）へ 302。`key&sig` は HMAC 検証後に提出物/zip へ 302（越権防止）。
- `_route`：
  1. **未登録/登録中** → `business.handle_registration`（氏名照合）
  2. テキストコマンド分岐：`テンプレ`（ボタン）/`一覧`（状況）/`一括DL`（zip）/`履歴`（DL付き）/`名簿系`（CRUD）→ 残りは `_route_text`
  3. **ファイル（Excel）** → `_handle_file`
- `_route_text`：保留ファイルの種別確定 / メニュー / `未提出確認` / `催促`。
- 人事専用コマンドは先頭で `business.is_hr(uid)` を判定。

## 業務：`common/business.py`
- **名簿（主データ）**：`roster_scan / roster_get / roster_find_by_name / roster_add / roster_resolve / roster_update_field / roster_delete / _next_emp_id(E###)`。
- **登録（氏名のみ）**：`handle_registration` … 初回は employees 行を `awaiting_name` で作成 → 氏名送信時に `roster_find_by_name`。一意一致なら employees に empId/name/active、roster に lineUserId を書込み。0件→`not_in_roster`、複数→`dup_name`。
- **権限**：`is_hr`（roster.role==hr ＋ HR_USERIDS）、`roster_of(user_id)`、`emp_name(user_id)`。
- **提出**：`save_submission` … 既存有無で `resubmit` 判定 → `s3util.put_submission`（氏名で命名）→ `record_submission`。`resolve_pending` は暫定置き→転正。`is_late_resubmit(period)= current>period`。
- **差分**：`submitters(period,type)`（GSI1）→ `missing / missing_all_types / roster_status`。roster 全員と提出者集合の差。未連携者は `linked=False`。
- **時刻**：`current_period()=JST yyyymm`、`normalize_period()` は `[202606]`/`6月`/`2026-06` を解釈、無ければ当月。

## LINE 適配：`common/line.py`
- `verify_signature` = base64(HMAC-SHA256(channel_secret, raw_body)) と `X-Line-Signature` を比較。
- `parse_events` → `{fromUser:'line:'+userId, msgType, content, messageId, fileName, event, replyToken}`。
- `reply / reply_messages / push`（api.line.me）、`download_content`（api-data.line.me で Excel 取得）。
- `buttons_message / carousel_message`：長URLをボタンに隠す（様式・履歴・一括DL）。

## S3：`common/s3util.py`
- `submission_key(period,type,name)` → `hr/{yyyy}/{mm}/{folder}/{label}（氏名）_YYYYMM.xlsx`。
- `put_submission`（ContentType=xlsx＋Tagging=lifecycle=managed）、`copy_to_submission`（暫定→転正）。
- `presign_get`（ResponseContentType/Disposition で iOS の .txt 化を防止、RFC5987 で日本語名）、`presign_template`。
- `build_month_zip(period)` … `hr/{yyyy}/{mm}/` を走査し zip 生成 → `exports/{period}.zip`。

## 提出フロー（例）
```
社員が Excel 送信
  └ webhook: infer_type(fileName)
       ├ 判別OK → save_submission → S3保存＋DynamoDB記録 → 「✅ 受け付けました」
       │           └ 締め後の再提出なら _maybe_notify_resubmit → 人事全員へ通知
       └ 判別NG → stash_pending → 「勤怠？経費？」→ 返信で確定 → resolve_pending
```

## リマインド：`handlers/reminder.py`
```
period = payload.period or 当月
targets = missing_all_types(period)（または type 指定）
for t in targets:  if t.lineUserId: messaging.send(push)  else: skip（未連携=人事が個別連絡）
```

## CDK：`lib/brightstar-hr-stack.ts`
- S3（BlockPublic/SSE/バージョン管理、ライフサイクル：pending 7日・exports 3日・提出物 60日→DeepArchive/365日削除）＋様式 BucketDeployment。
- DynamoDB ×3（roster/employees/submissions＋GSI1、On-Demand、暗号化、PITR）。
- Lambda ×2（webhook=Function URL、reminder=EventBridge cron）、最小 IAM、SSM 読取、env 注入。
