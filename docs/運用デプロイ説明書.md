# BrightStar 社内システム ｜ 使用説明書（デプロイ・運用・トラブル対応）

IaC は **AWS CDK (TypeScript)**。Lambda は **Python 3.12 / arm64（依存ゼロ）**。リージョンは東京（ap-northeast-1）。

---

## 1. 事前準備
- Node.js 18+、AWS CLI（東京リージョンで認証済み）
- LINE 公式アカウント（Messaging API）の Channel secret / access token

## 2. 初回デプロイ
```bash
cd brightstar-hr/cdk
npm install
npx cdk bootstrap aws://<account>/ap-northeast-1     # 初回のみ
```
LINE 資格情報を SSM(SecureString) に登録（**git に残さない**）：
```bash
aws ssm put-parameter --name /brightstar-hr/dev/line/secret --type SecureString --value '<channel secret>' --region ap-northeast-1
aws ssm put-parameter --name /brightstar-hr/dev/line/token  --type SecureString --value '<access token>'   --region ap-northeast-1
```
様式 Excel を `brightstar-hr/templates/`（`作業時間記録簿.xlsx` / `経費.xlsx`）に配置 → デプロイ：
```bash
npx cdk diff
npx cdk deploy        # 出力 LineWebhookUrl を控える
```

## 3. LINE 側設定
- Messaging API > **Webhook URL** に出力の `LineWebhookUrl` を設定 → **Verify** → **Use webhook ON**。
- 応答メッセージ（自動応答）・あいさつメッセージは **OFF**。

## 4. 初期データ（社員名簿）
- 名簿（roster テーブル）に社員を登録。少量なら人事が LINE から `社員追加 氏名 部署 [hr]` で投入可。
- 一括投入はスクリプト／コンソールで roster テーブルに put（`empId, name, department, role, lineUserId?`）。
- 人事担当者は `role=hr`。

## 5. 日常運用
- 社員：Excel 送信で提出（再提出可）、`テンプレ`/`履歴`。
- 人事：`未提出確認` / `一覧` / `催促` / `一括DL` / 名簿 CRUD。
- 自動リマインド：既定 毎月 25・28 日 09:00 JST（`cdk.json` の `reminderCron` で変更、`cdk deploy -c reminderCron="cron(...)"`）。

## 6. 変更デプロイ
- コード/構成を変更したら `npx cdk diff` → `npx cdk deploy`。Lambda 資産は自動再アップロード。

## 7. トラブル対応
| 症状 | 原因・対処 |
|---|---|
| LINE Verify が失敗 | Webhook URL の誤り。出力 `LineWebhookUrl` を再確認（末尾スラッシュ含め一致）。 |
| 送信しても無反応 | webhook ログ（CloudWatch `/aws/lambda/<app>-<stage>-webhook`）を確認。署名NG=403、resubmit/CRUD は print ログあり。 |
| 署名で 403 | SSM の secret 不一致。資格情報を更新後、**Lambda を再起動**（`update-function-configuration --description ...` で別バージョン化。`--environment` は使わない＝全 env が消えるため）。 |
| iOS で .txt 保存 | 旧不具合。対策済（presign で ContentType/Disposition 強制）。再発時は s3util の presign を確認。 |
| ボタンが届かない | LINE のボタン URL は 1,000 文字上限。提出物は必ず短縮リンク `/dl?key=&sig=` 経由（直接の署名付きURLは不可）。 |
| 2 か月超のファイルがDL不可 | Deep Archive へ移行済み。復元（restore）が必要。 |
| 名簿に無いと言われる | 氏名表記が roster と不一致。`社員変更`/`社員追加` で調整。 |

## 8. 監視・ログ
- CloudWatch Logs：`<app>-<stage>-webhook` / `<app>-<stage>-reminder`。
- リマインド結果は reminder ログに `sent` / `skipped(unlinked)` を出力。

## 9. クリーンアップ（dev/staging）
- 非 prod はデータリソースが `DESTROY`／S3 autoDelete のため `npx cdk destroy` で削除可。
- prod は `RETAIN`。SSM パラメータは別途 `aws ssm delete-parameter` で削除。
- **削除前に必ず内容確認**（提出物・名簿は重要データ）。
