# 要員通勤コスト比較ツール システム要件書

本書は、本ツールの機能要件・非機能要件・前提条件をまとめたものです。

---

## 1. 目的・適用範囲

社内の営業提案において、要員（社員）を客先現場へアサインする際の **通勤コスト比較** を支援する。
対象は社内利用（要員十数名規模・月数百回程度の問い合わせ）。

---

## 2. 機能要件

| ID | 要件 |
|----|------|
| FR-01 | 出発駅・到着駅から、所要時間・乗換回数・片道運賃（IC/きっぷ）・通勤定期代（1/3/6ヶ月）を取得する |
| FR-02 | 経路選択戦略（定期最安／時間最短）を指定して 1 経路を選定する |
| FR-03 | 現場を指定し、待機中（available）の全要員を一括比較する。確定済み（assigned）は除外する |
| FR-04 | 駅名の表記ゆれ（漢字/かな/カタカナ/ローマ字/中国語）を吸収して解決する |
| FR-05 | 住所から最寄駅を判定する（保存は駅名のみ） |
| FR-06 | 社員の登録・更新・削除、および CSV/txt による一括反映を行う |
| FR-07 | LINE・コマンドライン・API の 3 経路から同一の比較を実行できる |
| FR-08 | 比較結果を CSV 出力できる |

---

## 3. 非機能要件

| 区分 | 要件 |
|------|------|
| 性能 | キャッシュ命中時は即時応答。未取得区間は外部取得のため数十秒許容（非同期で返信） |
| 可用性 | サーバーレスでマネージド可用性に依拠。常時起動は不要 |
| コスト | 従量課金。低トラフィック前提でアイドル時はほぼ無課金 |
| セキュリティ | API は IAM 認証。LINE は署名検証。認証情報は SSM SecureString 管理（コード非格納） |
| 個人情報 | 社員住所は保存せず最寄駅のみ保持 |
| 保守性 | データ取得元を抽象化し差し替え可能。業務層は無改修で移行可能 |
| 礼節 | 外部データ取得は直列・低頻度・間隔スリープ・30日キャッシュ |

---

## 4. データ要件

| データ | 保持先 | 主キー | 備考 |
|--------|--------|--------|------|
| 社員 | DynamoDB | 社員ID | 社内システム花名册と同一キー。最寄駅・状態を保持 |
| 通勤キャッシュ | DynamoDB | 区間キー | 30 日 TTL で自動失効 |
| 駅マスタ | 同梱ファイル | 駅名 | 駅名→駅ID 対照。首都圏を収録 |

---

## 5. 実行・構築環境

| 区分 | 内容 |
|------|------|
| クラウド | AWS（東京リージョン ap-northeast-1） |
| 実行基盤 | AWS Lambda（Python 3.12）／ Amazon DynamoDB |
| 外部連携 | LINE Messaging API、Amazon Bedrock、Amazon Location Service |
| IaC | AWS CDK（TypeScript） |

---

## 6. 前提・制約

- 取得値は取得時点のダイヤに依存し変動する（目安値）。
- 暫定のデータ取得元を利用しており、正式運用では認可 API への切替を前提とする。
- LINE 公式アカウント側で「応答メッセージ OFF・応答モード Bot」の設定が必要。
- 簡体字中国語の解決は AI 変換に依存するため、まれに失敗し得る（日本語駅名で再送）。
